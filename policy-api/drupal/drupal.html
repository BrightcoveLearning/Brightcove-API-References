Policy API Reference

Reference for the Brightcove Policy API, used to create policy keys to access the Playback API. This API is used to generate keys to access the Playback API outside the context of a Brightcove Player.

Notes:

  • Policy keys are generated automatically for Brightcove Players.
  • To test API requests, you can use our API Testing Tools

Create a new policy key to access the Playback API

https://policy.api.brightcove.com/v1/accounts/:account_id/policy_keys

Header

Field Type Description
Content-Type String

Content-Type: application/json
Authorization String

Authorization: Bearer access_token (see Getting Access Tokens)

Path Parameters

Field Type Description
account_id String

Video Cloud account ID

Request Body Fields

Field Type Description
key-data Object

Data for the policy key
  account-id String

Video Cloud account id
  apis optional String[]

Array of APIs that are permitted for this key (currently "search" is the only one available - this must be included to use the search functionality for the Playback API)
  allowed-domains optional String[]

For domain restriction, the domains this key will work on
  require-ad-config optional Boolean

Whether Playback API requests require an ad-config-id URL parameter for server-side ad insertion

Default value: false
  • Create Policy Request Body Examples: 
{
  "key-data": {
      "account-id": "1752604059001",
      "require-ad-config": true,
      "apis": [
                "search"
              ],
     "allowed-domains": [
       "http://support.brightcove.com",
       "https://solutions.brightcove.com"
     ]
  }
}

Response Fields

Field Type Description
key_string String

The policy key string
key-data Object

Map of key data prescribing the policy
  account-id String

The Video Cloud account id
  apis Sting[]

Array of apis permitted for the key
  allowed-domains Sting[]

Array of domains allowed to use this key
  require-ad-config Boolean

Whether Playback API requests require an ad-config-id URL parameter for server-side ad insertion

Default value: false
  • Success Response: 
HTTP/1.1 200 OK
{
    "key-string": "BCpkAD123456789AbcdEFghIjklmnOP",
    "key-data": {
        "account-id": "1752604059001",
        "require-ad-config": true,
        "apis": [
            "search"
         ],
        "allowed-domains": [
            "http://support.brightcove.com",
            "https://solutions.brightcove.com"
         ]
     }
}

Error 4xx

Field Type Description
UNAUTHORIZED json

401: Authentication failed; check to make sure your policy key is correct
RESOURCE_NOT_FOUND json

404: The api couldn't find the resource you requested
NOT_AVAILABLE json

400: The resource you are requesting is temporarily unavailable

Error 5xx

Field Type Description
UNKNOWN json

500: Issue in Brightcove system; try again later.
TIMEOUT json

500: Server likely too busy; try again later.
  • 404 Error Response 
HTTP/1.1 404 Not Found
[
    {
        "error_code": "RESOURCE_NOT_FOUND"
    }
]
